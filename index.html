왜 사전 테스트가 필요한가?

"교육장에서는 되는데 사무실에선 안 된다"는 기술 클레임을 0%로 만들기 위한 작업이에요. 홈앤서비스는 ① 사내 보안정책으로 외부 AI Tool 제약 ② 내부 데이터 업로드 제한 ③ Excel 기반 업무가 특징이라, 교육 시작 D-14 ~ D-7 기간 안에 실제 교육용 PC/계정으로 모든 실습 플로우를 1회 리허설해야 해요.
우선순위 표기: ■ 높음 (막히면 교육 진행 불가) · ■ 중간 (대체 플랜 있으면 OK) · ■ 낮음 (있으면 좋음)

① Smart Work 과정 (1일, 7H)

자동화 Tool 3H VBA 4H 초급-중급 이론 30% / 실습 70%

과정 재구성 요지: 사내 AI Tool 제약 + 내부 데이터 업로드 제한 + Excel 업무 중심 환경을 고려해, 자동화 Tool 이해(n8n / Make / Dify) 3H + VBA 이해 & ChatGPT 활용 구현 4H로 구성. VBA 코딩을 가르치는 게 아니라 "ChatGPT로 자동화 설계 → 사내 환경에 맞게 구현"하는 흐름.

웹 접근성 n8n.io, make.com, dify.ai 사내망 접속 가능 여부 교육용 PC에서 Chrome/Edge로 3개 사이트 접속 → 회원가입 페이지까지 진입 확인.
차단 시 IT팀에 방화벽 예외 신청 (IP/도메인 화이트리스트) 		높음
계정 생성 외부 이메일 기반 가입 허용 여부 / 회사 이메일 사용 가능 여부 테스트 계정 1개를 사내 Gmail/회사 이메일로 생성 시도.
무료 플랜 한도 확인 (n8n Free: 월 5,000 실행 / Make Free: 월 1,000 operation) 		높음
트리거 연동 Gmail, Outlook, MS Teams, Google Sheets 커넥터 OAuth 인증 통과 n8n/Make에서 Gmail → Sheets 샘플 워크플로 1개 만들어 실제 인증까지 통과.
사내 Outlook 계정은 Microsoft Graph API 승인 필요 여부 확인. 		높음
OpenAI 노드 AI 노드(ChatGPT/Claude) API Key 발급 & 사용 가능 여부 교육용 OpenAI API 키 1개 발급 → Make의 OpenAI 모듈로 "Hello World" 응답 호출.
결제 카드 등록·월 사용량 한도 설정 (교육 1회당 약 $10 기준). 		높음
더미데이터 실습용 더미 VOC/AS접수/거래처 데이터셋 준비 홈앤서비스 실제 데이터 구조(컬럼명·타입) 참고해 비식별 더미 100~300행 제작.

파트 2. VBA 이해/실습 (4H) — ChatGPT 활용 설계 중심

카테고리 무엇을 테스트? 어떻게 테스트? 우선순위
매크로 활성화 사내 PC Excel에서 VBA/매크로 실행 가능 여부 Excel → 옵션 → 보안센터 → 매크로 설정 확인.
"모든 매크로 제외" 정책이면 IT팀에 교육용 PC 예외 요청.
xlsm 파일 저장 가능 여부도 체크 (일부 환경은 차단). 		높음
개발 도구 VBA 편집기(Alt+F11) 접근 가능 여부 Excel 리본 "개발 도구" 탭 노출 → Visual Basic Editor 실행 → 표준 모듈 1개 생성 테스트. 높음
신뢰할 수 있는 위치 실습 파일 저장 경로가 매크로 실행 허용 위치인지 C:\Users\...\Documents 같은 표준 경로에 xlsm 저장 후 매크로 실행 테스트.
사내 공유 드라이브·원드라이브 경로에서는 실행 차단될 수 있어 로컬 경로 확보. 		높음
실전 시나리오 4종 교육에서 다룰 4가지 자동화 시나리오 실제 작동 확인:
  • 여러 시트 통합
  • 데이터 자동 정리(빈칸/형식)
  • 중복 제거 및 요약
  • 버튼 기반 보고서 정리
 각 시나리오별 더미 xlsx 준비 → ChatGPT에 "이 구조로 ~~해줘" 프롬프트 → 받은 VBA 코드 붙여넣기 → 실행 → 정상 동작까지.
사내 Excel 버전(2016/2019/365) 호환성 확인 — Power Query, Dynamic Array 함수 사용 가능 여부. 		높음
ChatGPT 접근 사내망에서 ChatGPT 웹 접속 / API 사용 가능 여부 chat.openai.com 접속 → 로그인 → 샘플 질문 응답 확인.
차단 시: Copilot(MS 365), 사내 승인 LLM 대체 검토. 		높음
ActiveX 컨트롤 Excel 시트에 버튼/양식 컨트롤 삽입·매크로 연결 개발도구 → 삽입 → 버튼 → 매크로 할당 → 클릭 시 실행 테스트.
"버튼 기반 보고서 정리" 실습의 필수 요소. 		높음
참조 라이브러리 Scripting.Dictionary, FileSystemObject, ADO 등 기본 참조 사용 가능 VBE → 도구 → 참조 → Microsoft Scripting Runtime 체크 가능 확인.
중복제거/시트통합 실습에 자주 쓰임. 		중간
참여자 데이터 적용 참여자 본인 업무 데이터 → 실습 포맷 적용 흐름 사전과제: 참여자에게 "본인이 반복 사용하는 Excel 파일 1개" 지참 안내.
개인정보/민감정보 비식별 가이드 사전 공유. 		중간
핵심 리스크: VBA 매크로가 그룹 정책으로 막혀있을 가능성이 가장 큼. D-14 시점에 IT팀과 교육용 PC 10~30대 대상 매크로 활성화 예외 신청 공문 필요. 못 막을 거면 Power Automate Desktop 대체 시나리오도 Plan B로 준비.

② SQL with AI 과정 (1일, 7H)

데이터 추출→가공→시각화→공유 E2E 초급-중급 이론 30% / 실습 70%

흐름: AI 기반 데이터 업무 진단(0.5H) → AI 쿼리 생성(2H) → Power Query 가공(1.5H) → 시각화(0.5H) → 보고서 공유(1H) → 업무 적용 설계(1.5H)

카테고리 무엇을 테스트? 어떻게 테스트? 우선순위
MS SQL 접속 교육용 MS SQL Server 접속 / 읽기 권한 확보 SSMS(SQL Server Management Studio) 또는 Azure Data Studio 설치 확인.
교육용 별도 DB(실서비스 분리) 구축 → 비식별 샘플 테이블(AS접수/고객/상품 등) 적재.
읽기 전용 계정 발급 → SELECT 쿼리 실행 확인. 		높음
샘플 데이터 현업 구조 반영 비식별 샘플 DB 준비 홈앤서비스 실제 테이블 스키마 3~5개 복제 → 익명화된 더미 10,000~100,000행 INSERT.
JOIN / GROUP BY / 날짜 조회 등 실습에 필요한 필드 구성 확인. 		높음
Power Query 연결 Excel → Power Query → MS SQL 직접 연결 가능 여부 Excel 데이터 탭 → 데이터 가져오기 → SQL Server → 서버/DB 입력 → 인증 → 테이블 로드 성공까지.
사내 네트워크에서 DB 포트(보통 1433) 열려있는지 IT팀 확인.
"새로 고침" 자동화 스케줄 구동 여부도 체크. 		높음
AI 쿼리 생성 ChatGPT/Copilot에게 자연어→SQL 변환 요청 결과 품질 "이번 달 지역별 AS 건수 TOP10" 같은 샘플 요청 10개 → 생성된 쿼리 실제 DB에서 실행 → 정답률 확인.
실패 패턴(테이블명 허위 생성 등) 사전 파악 → 강의 중 다룰 "검증 체크리스트" 업데이트. 		높음
쿼리 라이브러리 사내 공유용 쿼리 저장소(Notion/Confluence/SharePoint) 접근 학습자가 만든 쿼리 저장·재사용 공간 1곳 선정 → 권한 확인 → 템플릿 페이지 준비. 중간
시각화 Excel 동적 차트 / Power Query 연동 차트 작동 SQL→Excel 쿼리 새로고침 시 차트 자동 갱신되는지 확인.
조건부 서식·슬라이서·피벗 차트 사용 제한 없는지 체크. 		중간
AI 보고서 초안 보고서 데이터 → AI에게 요약 요청 → 이메일 초안 생성 흐름 사내 허용 AI 도구(ChatGPT/Copilot/Gemini 중)로 샘플 수치 붙여넣기 → 요약·이메일 초안 결과물 품질 확인.
데이터 업로드 제한이 있다면 "수치만 복사 붙여넣기" 워크플로 확정. 		높음
메일 발송 Outlook에서 초안 붙여넣기 → 발송 정상 동작 사내 메일 서버로 실제 발송 테스트 (수신자: 테스트 계정 1개).
첨부파일(xlsx) 사이즈 제한 확인. 		낮음
핵심 리스크: Power Query → MS SQL 직접 연결이 네트워크 정책으로 막히는 사례 多. 막히면 사전 추출된 CSV/xlsx 기반 실습으로 Plan B 전환. 단, "E2E 리드타임 단축" 메시지가 약해지니 최대한 직결 확보.
