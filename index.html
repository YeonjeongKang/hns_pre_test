높음

  6. 똑같이 dify.ai 입력 후 엔터 → 홈페이지 로드 확인.
✓ 성공: 3개 사이트 모두 홈페이지가 정상적으로 보이고, 이미지/글자가 깨지지 않았다.
✗ 실패 증상:
• "이 사이트에 연결할 수 없음" / "액세스가 거부되었습니다" 문구가 뜬다
• 계속 동그라미만 돌고 화면이 안 뜬다 (1분 이상)
• 회사 차단 안내 페이지가 나온다
IT팀 보고 문구: 안녕하세요. 4월 XX일 홈앤서비스 AI 교육 준비로 아래 사이트 접속이 필요한데 현재 차단된 것으로 보입니다. 교육용 PC에 대한 화이트리스트 추가 부탁드립니다. - n8n.io - make.com - dify.ai - app.n8n.cloud - www.make.com 사용 기간: 2026-XX-XX ~ 2026-XX-XX (교육 포함 전후 1주일)

높음
왜 확인? 회원가입이 안 되면 실습에서 계정을 만들 수 없어요. 회사 이메일로 가입 가능한지 먼저 확인해야 해요.

✓ 성공: 회사 이메일로 가입이 되고, 로그인해서 워크플로 만드는 메인 화면까지 진입했다.

IT팀 보고 문구: n8n/make에서 보내는 인증 메일이 스팸 차단되는 것 같습니다. noreply@n8n.io, noreply@make.com 발신 도메인 수신 허용 요청드립니다.

  4. 회사 이메일 계정 로그인 화면이 뜨고, 권한 허용 버튼까지 누를 수 있는지 확인.
  5. "연결됨" 표시가 뜨면 성공.

• 로그인 팝업이 안 뜨거나 즉시 닫힌다
• 권한 허용 후에도 "연결 실패" 뜬다
IT팀 보고 문구: Microsoft 365 테넌트에서 n8n/make 앱의 OAuth 접근이 관리자 승인 필요 상태입니다. 교육용으로 아래 앱에 대한 Admin Consent 부여 요청드립니다. - n8n (App ID는 IT팀이 Azure AD에서 확인 가능) - Make.com
1-4. OpenAI API 키 발급 & n8n 연동
높음 기술자 필요
왜 확인? n8n에서 "AI가 메일 분류해줘" 같은 노드를 쓰려면 OpenAI 키가 필요해요. 키 발급과 호출까지 되는지 미리 확인.
준비물: 강사 또는 엔지니어 1명, 교육용 결제카드, OpenAI 계정
스텝 (동반자가 해줄 내용)
  1. platform.openai.com 로그인 → API Keys 메뉴 → Create new secret key로 키 1개 발급.
  2. Billing 메뉴에서 카드 등록 + $20 선충전 + 월 한도 $50으로 제한.
  3. n8n 워크플로에 OpenAI 노드 추가 → 방금 발급한 키 붙여넣기.
  4. "Hello, world"라고 입력하고 실행 → 응답이 오면 성공.
✓ 성공: n8n 실행 후 "Hello! How can I help you?" 같은 응답이 돌아온다.
✗ 실패 증상:
• "API key invalid" → 키 복사 실수
• "Request timed out" → api.openai.com 네트워크 차단
• "Insufficient quota" → 결제 미등록
IT팀 보고 문구: api.openai.com (OpenAI 호출 엔드포인트) 네트워크 접속 확인 부탁드립니다. 교육용 PC에서 Timeout 발생 중입니다.
1-5. 실습용 더미 데이터 준비 확인
중간
왜 확인? 교육에서 사용할 샘플 VOC/AS/거래처 데이터가 교육용 PC에 미리 있어야 실습이 원활해요.

  2. 각 파일을 더블클릭해서 Excel로 정상 열리는지 확인.
  3. 파일 안에 실제 개인정보(주민번호, 전화번호, 이름)가 없는지 눈으로 확인. 있으면 즉시 강사에게 알려 교체.
✓ 성공: 모든 더미 파일이 열리고, 개인정보 없이 실습 가능한 상태.
✗ 실패 증상: 파일이 없거나, Excel 열 때 "손상된 파일" 경고 뜸.

파트 2. VBA 이해/실습 (4H) — ChatGPT로 설계 중심

2-1. Excel 매크로 실행 권한 확인 (가장 중요!)
높음
왜 확인? 사내 보안정책으로 매크로가 차단되어 있으면 VBA 실습 자체가 불가능해요. 이 하나가 막히면 VBA 4시간 통째로 못 함.
준비물: 교육용 PC, Microsoft Excel
스텝
  1. Excel을 실행하세요 (빈 통합문서 새로 만들기).
  2. 좌측 상단 파일 메뉴 → 맨 아래 옵션 클릭.
  3. 왼쪽 메뉴에서 보안 센터 클릭 → 오른쪽 보안 센터 설정 버튼 클릭.
  4. 왼쪽 메뉴에서 매크로 설정 클릭.
  5. 현재 어떤 항목에 체크되어 있는지 확인. 아래 중 하나여야 합니다:
        ○ "알림이 없는 매크로 사용 안 함" → 실패 (가장 안 좋은 상태)
        ● "알림이 있는 매크로 사용 안 함" → 조건부 가능 (사용자가 매번 "활성화" 클릭)
        ● "모든 매크로 사용" → 성공
  6. 체크 항목을 바꿔볼 수 있는지 시도해보세요. 바꿀 수 없게 비활성화(회색)되어 있으면 그룹 정책으로 잠긴 것. IT팀만 풀 수 있어요.
✓ 성공: "알림이 있는 매크로 사용 안 함" 이상의 설정이고, 필요시 변경 가능하다.
✗ 실패 증상:
• 매크로 설정 화면 자체가 안 보인다
• 라디오 버튼들이 회색으로 비활성화되어 클릭 안 된다
• "조직의 정책에 의해 관리됩니다" 안내 문구가 떠 있다
IT팀 보고 문구: 안녕하세요. 홈앤서비스 AI 교육(4/XX)에서 VBA 실습이 진행됩니다. 현재 교육용 PC에서 Excel 매크로가 그룹 정책으로 차단되어 있어 실습 진행 불가 상태입니다. 요청 사항: 1) 교육용 PC XX대 한정으로 매크로 설정을 "알림이 있는 매크로 사용 안 함"으로 변경 가능하도록 풀어주세요. 2) xlsm(매크로 포함 Excel) 파일 저장·실행 허용 요청. 3) 교육 기간(XX~XX) 종료 후 원복 예정입니다. 법인 보안 정책상 어려우시면 대체 안(예: 교육 전용 VM 제공) 협의 부탁드립니다.
2-2. 개발 도구 탭 & VBA 편집기 접근 확인
높음
왜 확인? VBA 코드를 붙여넣을 편집기(VBE)에 접근할 수 있어야 실습이 돼요.
준비물: Excel
스텝
  1. Excel 실행 → 리본 메뉴(맨 위 메뉴바) 확인.
  2. 개발 도구 탭이 있나요? 없으면:
        → 리본 빈 곳에 우클릭 → 리본 메뉴 사용자 지정 → 오른쪽 목록에서 개발 도구 체크 → 확인.
  3. 탭이 나오면 개발 도구 클릭 → 왼쪽에 Visual Basic 버튼 클릭.
  4. 새 창이 뜨면서 "Microsoft Visual Basic for Applications" 창이 열리면 성공.
  5. 창 안에서 삽입 → 모듈 클릭해서 빈 모듈이 하나 생성되는지 확인.
  6. 단축키 테스트: Excel로 돌아와서 Alt + F11 키 눌렀을 때도 같은 VBE 창이 뜨는지 확인.
✓ 성공: VBE 창이 열리고 모듈을 만들 수 있다.
✗ 실패 증상: Alt+F11이 아무 반응 없다 / "이 기능은 사용할 수 없습니다" 팝업.
IT팀 보고 문구: 2-1 항목과 함께 보고. (같은 원인일 가능성이 큼)
2-3. xlsm 파일 저장 & 실행 확인
높음
왜 확인? 매크로가 포함된 Excel은 xlsm 확장자로 저장해야 해요. 이게 막히면 작업물 저장 자체가 불가능.
준비물: Excel
스텝
  1. 새 Excel 파일 → 파일 → 다른 이름으로 저장.
  2. 저장 위치를 바탕화면 또는 내 문서로 선택.
  3. 파일 형식 드롭다운에서 Excel 매크로 사용 통합 문서(*.xlsm) 선택.
  4. 파일명 테스트.xlsm 입력 후 저장.
  5. 저장 성공 후 파일을 닫았다가 다시 더블클릭해서 열기.
  6. "보안 경고: 매크로가 사용하지 않도록 설정되었습니다"가 노란 바에 뜨면 콘텐츠 사용 클릭.
  7. 경고 없이 열리면 성공. 경고조차 없이 매크로 완전 차단이면 실패.
✓ 성공: xlsm으로 저장되고, 다시 열 때 "콘텐츠 사용" 버튼으로 매크로 활성화가 가능하다.
✗ 실패 증상:
• 저장할 때 "이 파일 형식은 허용되지 않습니다" 뜸
• 파일이 저장은 되지만 다시 열면 매크로 코드가 모두 사라져 있음
• "관리자 권한이 필요합니다" 팝업
2-4. 실습 시나리오 4종 실제 작동 리허설
높음 기술자 필요
왜 확인? 교육에서 다룰 실제 4가지 시나리오가 교육용 PC에서 처음부터 끝까지 작동하는지 강사와 함께 리허설.
준비물: 강사 1명, 시나리오별 더미 xlsx 파일 4개, ChatGPT 접속
스텝 (강사가 시연, 운영 담당자는 옆에서 관찰하며 기록)
  1. 시나리오 A - 여러 시트 통합: 10개 시트가 있는 xlsx 파일을 연다 → ChatGPT에 "이 구조에서 모든 시트를 하나로 합쳐주는 VBA 코드 만들어줘" → 받은 코드를 VBE에 붙여넣기 → 실행 → 통합 시트 생성 확인.
  2. 시나리오 B - 데이터 자동 정리: 빈칸/잘못된 형식이 섞인 파일을 연다 → "빈 행 삭제하고 날짜 형식 통일하는 코드 만들어줘" → 실행 → 결과 확인.
  3. 시나리오 C - 중복 제거/요약: 중복이 많은 거래 데이터 파일 → "거래처별 중복 제거하고 합계 만들어줘" → 실행 → 결과 확인.
  4. 시나리오 D - 버튼 기반 보고서: 개발도구 → 삽입 → 단추 → 시트에 그리기 → 매크로 연결 → 버튼 클릭 시 자동으로 보고서 서식이 적용되는지 확인.
  5. 4개 모두 오류 없이 끝까지 진행되는지 기록.
✓ 성공: 4개 시나리오 모두 오류 없이 결과물이 나온다.
✗ 실패 증상: 특정 시나리오에서만 오류 (버전 호환 이슈일 가능성). Excel 버전(2016/2019/365) 확인 후 강사에게 공유.
강사에게 전달: 시나리오 X에서 오류 발생했습니다. 오류 문구 스크린샷 첨부합니다. 사내 Excel 버전은 Microsoft 365 / 버전 2XXX입니다.

SQL-1. SSMS(SQL 관리 도구) 설치 및 MS SQL 접속
높음 기술자 필요
왜 확인? 학습자가 데이터를 직접 뽑아보려면 SQL 접속 도구(SSMS)가 설치되어 있고 교육용 DB에 연결되어야 해요.
준비물: 강사 또는 DBA 1명, 교육용 DB 서버 주소·계정 정보
스텝 (동반자와 함께)
  1. 윈도우 시작 메뉴에서 SSMS 또는 SQL Server Management Studio 검색.
  2. 있으면 실행. 없으면 강사/IT팀에 설치 요청.
  3. SSMS 실행 후 연결 창에 서버 주소 입력 (강사가 알려준 주소, 예: edu-sql.homenservice.local).
  4. 인증 방식 선택 → 사용자명/비밀번호 입력 → 연결.
  5. 왼쪽 창에 데이터베이스 목록이 보이고 교육용 DB(예: EDU_DB)가 있으면 성공.
  6. 쿼리 테스트: 새 쿼리 창 열고 SELECT TOP 10 * FROM dbo.AS접수샘플; 입력 후 F5 → 데이터 10행이 돌아오면 OK.
✓ 성공: SSMS로 접속되고 샘플 쿼리 결과가 나온다.
✗ 실패 증상:
• "서버 연결 실패" / "네트워크 경로를 찾을 수 없음" → 방화벽/포트 차단
• 로그인 실패 → 계정 정보 오류
• DB가 비어있음 → 샘플 데이터 미적재
IT팀/DBA 보고 문구: 교육용 PC에서 edu-sql.XXX.local:1433 포트 접속 시 timeout 발생합니다. - 교육용 PC의 IP 대역에서 해당 서버로의 1433 포트 인바운드 허용 요청 - 교육용 계정(edu_reader) 권한 및 비밀번호 재확인 요청
SQL-2. Excel Power Query로 SQL 서버 직접 연결
높음 기술자 필요
왜 확인? 이 과정의 핵심 메시지인 "SQL→Excel 자동 새로고침"을 시연하려면 Excel이 SQL 서버에 직접 연결돼야 해요.
준비물: 강사, Excel, SQL 서버 주소
스텝
  1. Excel → 상단 데이터 탭 클릭.
  2. 데이터 가져오기 → 데이터베이스에서 → SQL Server 데이터베이스에서 클릭.
  3. 서버 주소 입력 (예: edu-sql.homenservice.local), 데이터베이스 이름 입력 후 확인.
  4. 사용자명/비밀번호 입력 → 연결.
  5. 왼쪽 창에 테이블 목록이 보이면 테이블 1개 선택 → 로드.
  6. Excel 시트에 데이터가 불러와지면 성공.
  7. 추가 확인: 데이터 탭 → 모두 새로 고침 클릭 시 SQL에서 데이터 다시 불러오는지 확인.
✓ 성공: Excel에 SQL 데이터가 로드되고 새로고침이 작동한다.
✗ 실패 증상:
• "드라이버를 찾을 수 없습니다" → ODBC 드라이버 미설치
• 연결 시도 후 5분 이상 응답 없음 → 네트워크 차단
• 연결은 되지만 "권한 없음" → 계정 권한 부족
IT팀 보고 문구: 교육용 PC의 Excel에서 Power Query → SQL Server 직접 연결이 필요합니다. - Microsoft OLE DB Driver for SQL Server 설치 필요 (없을 경우) - 교육용 PC → edu-sql 서버 1433 포트 열기
SQL-3. 비식별 샘플 데이터 품질 확인
높음
왜 확인? 교육용 DB 안에 실습 가능한 데이터가 충분히, 그리고 개인정보 없이 들어있어야 해요.
준비물: SSMS로 접속된 상태, 강사가 제공한 샘플 데이터 명세서
스텝 (명세서 없으면 강사에게 "오늘 교육에서 쓸 테이블 목록 주세요" 요청)
  1. SSMS에서 교육용 DB 확장 → 테이블 폴더 확장.
  2. 명세서에 적힌 테이블(보통 3~5개: AS접수, 고객, 상품 등)이 모두 있는지 눈으로 확인.
  3. 각 테이블 우클릭 → 상위 1000개 행 선택.
  4. 결과 데이터에서 개인정보(실명, 주민번호, 실제 전화번호, 주소)가 없는지 10행 정도 눈으로 검토. 있으면 즉시 강사에게 교체 요청.
  5. 행 수가 교육 실습에 충분한지(보통 최소 1,000행 이상) 확인:
    SELECT COUNT(*) FROM 테이블명; 실행 후 결과 숫자 기록.
✓ 성공: 모든 테이블이 있고, 개인정보 없고, 1,000행 이상 있음.
✗ 실패 증상: 실명/전화번호 발견 / 행 수 부족 / 특정 테이블 없음.
강사/DBA 보고: X테이블에 실명으로 보이는 데이터가 있습니다. 비식별 처리 요청드립니다. Y테이블은 10행만 있어 실습에 부족합니다. 최소 1,000행 확장 요청.
SQL-4. AI 쿼리 생성 품질 리허설
높음 기술자 필요
왜 확인? "자연어로 질문 → AI가 쿼리 작성 → 실행"이 이 과정의 핵심 데모. 실제 테이블로 해봤을 때 생성된 쿼리 정답률을 체크해야 해요.
준비물: 강사, ChatGPT 또는 Copilot 접속, SSMS
스텝 (강사가 시연)
  1. ChatGPT에 테이블 스키마를 붙여넣고 "이 구조에서 OO 조회하는 쿼리 만들어줘" 요청.
  2. 생성된 쿼리를 SSMS에 붙여넣고 실행.
  3. 오류 없이 결과가 나오는지 확인.
  4. 이 과정을 샘플 질문 5~10개로 반복:
      • "이번 달 지역별 AS 건수 TOP10"
      • "고객 등급별 월 매출 합계"
      • "최근 30일간 특정 상품 판매 추이"
      • "불량률 상위 5개 제품"
      • "재구매율이 높은 고객 100명"
  5. 몇 개가 한 번에 성공했는지 기록 (예: 10개 중 7개 성공).
✓ 성공: 10개 중 6개 이상이 오류 없이 정답 근처로 나온다.
✗ 실패 증상: 없는 테이블명/컬럼명을 만들어낸다 (Hallucination). → 실패 사례는 강사에게 공유해서 강의 중 "검증 체크리스트" 예시로 활용.
SQL-5. Excel 동적 차트 자동 갱신 확인
중간
왜 확인? SQL→Excel로 가져온 데이터로 만든 차트가 데이터 새로고침 시 자동 업데이트되는지 검증.
스텝
  1. SQL-2에서 가져온 Excel 데이터 영역을 선택 → 삽입 탭 → 차트(막대/꺾은선) 추가.
  2. 차트가 정상 생성되는지 확인.
  3. SQL DB에 데이터 1행 추가 (강사 요청).
  4. Excel에서 데이터 → 모두 새로 고침 클릭.
  5. 차트에 새 데이터가 반영되는지 확인.
✓ 성공: 새로고침 시 차트가 자동으로 업데이트된다.
SQL-6. AI로 이메일 초안 생성 & Outlook 발송
높음
왜 확인? "보고서 → AI가 이메일 초안 작성 → 발송"의 마무리 단계가 사내 Outlook에서 정상 작동해야 E2E 흐름이 완성돼요.
준비물: ChatGPT 접속, Outlook 실행, 테스트 수신 이메일 1개
스텝
  1. Excel 시트에서 보고서 데이터 일부(표 하나)를 복사.
  2. ChatGPT에 붙여넣고 "이 데이터 기반으로 팀장에게 보내는 주간 보고 이메일 초안 써줘" 요청.
  3. 돌아온 초안을 Ctrl+C로 복사.
  4. Outlook 열고 새 메일 작성 → 제목·본문에 붙여넣기.
  5. 테스트 수신자(본인 이메일)에 발송 → 실제 도착하는지 확인.
✓ 성공: 메일이 정상 수신되고 서식이 깨지지 않는다.
✗ 실패 증상: 외부 이메일로 발송이 차단되거나, 본문 포맷이 깨짐.
⚠ SQL with AI 핵심 리스크: SQL-2(Power Query→MSSQL 직결)가 네트워크 정책으로 막히는 사례가 가장 많아요. 막히면 사전 추출된 CSV 기반 실습으로 전환 가능하지만, "E2E 리드타임 단축" 메시지가 약해지니 최대한 직결 확보 권장.
